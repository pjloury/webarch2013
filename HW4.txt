2.  	Explain what happens when your browser loads http://people.ischool.berkeley.edu/~<USER>/server/wiki 
When you enter into my browser the URL http://people.ischool.berkeley.edu/~pjloury/server/wiki
The browser queries the iSchool server people.ischool.berkeley.edu and accesses the wiki file in the server directory /~pljoury/server/wiki

Wiki contains the following HTML code:
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 3.2 Final//EN">
<title>Redirecting...</title>
<h1>Redirecting...</h1>
<p>You should be redirected automatically to target URL: <a href="http://en.wikipedia.org">http://en.wikipedia.org</a>.  If not click the link.

The server then handles a re-direct request to http://en.wikipedia.org
and outputs the following:

Request 
DEBUG in app [./app.py:38]:
Redirecting to http://en.wikipedia.org
--------------------------------------------------------------------------------
127.0.0.1 - - [23/Sep/2013 14:25:00] "GET /wiki HTTP/1.1" 302 -

The app.py server then submits a GET request to Wikipedia.org and the Wikipedia.org server then sends back the HTML page for en.wikipedia.org, and this is the HTML code that is then displayed in the browser


3.	127.0.0.1 - - [23/Sep/2013 14:16:59] "GET /~pjloury/server" 404 –

For this given route, no HTML page is configured to be rendered and so the server returns a 404 error because the user is attempting to access something that is not there.
